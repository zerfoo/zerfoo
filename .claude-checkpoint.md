### Root Cause of Remaining Quality Issue:
The GPU Q5_0 path pre-dequantizes to F32 and physically transposes before cuBLAS Sgemm.
This loses the block-level numerical structure that the CPU mmap path preserves via
GemmF32MmapNT (which reads Q5_0 blocks and accumulates scale*sum(qi*xi) as a single
FMA chain). The per-element pre-dequantization introduces ~0.088 logit error through
18 layers, which changes the argmax at the first token.
